Smart Parking Management System

🚗 Smart Parking Management System — Flask-Based Web Application

The Parking Management System is a dynamic web application built with Flask (Python) that enables users to manage vehicle entries and exits within a parking lot efficiently. It replicates a real-world parking workflow through a minimalistic UI and precise backend logic. Designed with both usability and scalability in mind, this system offers essential functionality such as tracking parking sessions, generating unique identification codes, and computing time-based charges — all from a simple web interface. This system is ideal for small-scale parking lots, demo environments, or educational purposes, and it serves as a foundation for integrating more complex features like admin panels, QR code tracking, or database persistence.

✅ Core Features

  • 🔐 Vehicle Session Tracking: Records vehicle number, type, unique session code, and entry time.
  • 🧾 Unique Code Generation: Each vehicle is assigned an 8-character unique code using Python’s uuid for session identification.
  • 🕒 Duration-Based Billing: Charges are calculated at a fixed per-minute rate using the difference between entry and exit timestamps.
  • 💬 Real-Time User Feedback: Clear success or error messages displayed upon vehicle entry and exit attempts.
  • 💾 In-Memory Session Handling: Active parking sessions are stored in a Python list for simplicity and fast prototyping.
  • 📊 MySQL-Compatible Schema: Includes a ready-made SQL table design (ParkingRecords) for future integration with relational databases.
  • 🧩 Modular Flask Codebase: Clean separation of routes, logic, and HTML templates, making it easy to upgrade or scale.
  • 🎨 Minimalist Frontend: Simple, responsive HTML pages rendered via Flask and Jinja2.
  • ⚙️ Error Handling: Gracefully handles incorrect inputs, duplicate exits, or unmatched codes.
  • 🚀 Lightweight & Deployable: Can run locally or on any basic cloud environment with minimal setup.

Technologies Used:

  • Backend: Python, Flask
  • Frontend: HTML5, Jinja2 (templating)
  • Libraries/Utilities:
    • uuid (for session code generation)
    • datetime/timezone (for duration calculations)
  • Database: MySQL
  • Dev Environment: VS Code, PowerShell, Python 3.12

📌 Use Case Scenarios

  • Local businesses needing basic parking management
  • Academic demonstration of Flask project structure
  • Freelance or internship projects showcasing backend capabilities
  • Prototype for larger parking automation systems

Banking Management System

🏦 Console-Based Python Banking System

The Bank Management System is a Python-based CLI (Command Line Interface) application designed to simulate real-world banking operations through an interactive and secure terminal experience. Built using Python and MySQL, the system offers a solid foundation for managing basic banking functionalities such as opening accounts, deposits, withdrawals, balance inquiries, and updating personal details.

✅ Core Features

  • 🔐 Account Management

    Securely creates and manages user accounts with unique account numbers, ensuring that each customer’s data remains distinct and easily retrievable.

  • 💵 Deposit & Withdraw functionalities with basic validation
  • Cash Withdrawal & Deposit

    Handles real-time balance updates by allowing users to deposit or withdraw money. Ensures integrity by checking for sufficient funds before processing a withdrawal.

  • 📄 Account Statement

    Displays user account information such as account number, holder’s name, balance, and registered mobile number upon verification.

  • 📲 Update Account Details

    Users can update their account holder’s name or registered mobile number through secure validation of existing credentials.

  • 🔄 Auto Account Number Generation

    The system automatically generates sequential account numbers based on the last existing entry in the database, removing the need for manual tracking.

  • 🔐 Data Validation & Error Handling

    Includes input validation for account details and handles exceptions like invalid account number, name mismatch, or insufficient funds.

  • 💾 Persistent Storage with MySQL

    All account records are stored in a MySQL database, ensuring data persistence, easy querying, and scalability for larger datasets.

  • 🧩 Extensible Python Codebase

    Cleanly structured code with logical separation of tasks, making it easy to extend the functionality or port to a GUI/web version in the future.

  • 🖥️ Console-Based UI

    Minimalistic, clear prompts and feedback messages create a user-friendly command-line experience, ideal for educational or backend prototyping use cases.

Technologies Used:

  • Database: MySQL 8+
  • Libraries/Utilities:
